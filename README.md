# 幕布 → Obsidian 交易笔记迁移与知识库系统化

> 从 proprietary 笔记软件到本地 Markdown 知识库的完整迁移与整理流程

---

## 背景

### 原始状态

- **笔记软件**：幕布（Mubu）— 国产大纲式笔记应用，Electron 架构
- **笔记数量**：28 篇交易相关笔记
- **笔记主题**：期货/期权交易（ES、NQ），涵盖拍卖市场理论、订单流分析、GEX 指标、交易错误复盘
- **存储格式**：幕布私有二进制格式（SQLite + LevelDB + Blob 存储），本地路径 `%APPDATA%\Roaming\Mubu\mubu_appdata\`
- **痛点**：无法通过 API 访问、本地数据不可直接读取、与 AI 工具协作困难

### 目标

将全部交易笔记迁移至 **Obsidian**（本地 Markdown 知识库），实现：

1. **可读取** — AI 工具可直接访问 `.md` 文件
2. **可编辑** — AI 可修改、增强、关联笔记
3. **可检索** — 全文搜索 + 双向链接网络
4. **离线可用** — 不依赖任何外部服务

## Phase 4: 知识库内容分析

### 知识体系全景

#### 领域 A: 核心交易方法论（hc_2.0）

**理论基础**：拍卖市场理论（Auction Market Theory）+ 订单流分析（Order Flow）

**核心概念三角**：
```
        吸收 (Absorption)
       /          \
      /            \
  失衡              突破
(Imbalance)    (Breakout)
      \            /
       \          /
        衰竭 (Exhaustion)
```

**跨概念出现频率**：
- 吸收 — 5 篇提及
- 失衡 — 5 篇提及
- Delta — 4 篇提及
- 大单 — 3 篇提及
- 假突破 — 3 篇提及

**最完整笔记**：突破交易、吸收与拍卖、拍卖失衡、支撑与阻力的假突破

**待深化笔记**：ES and NQ（仅 4 行）、关键拍卖反转（碎片化）

#### 领域 B: 错单总结

**分级体系**：
- **L1**（最强）：期权 + 结构 + 时间三维共振（待补充具体模式）
- **L2**（强）：价格跨度超级大的压单群_反转
- **L3**（弱，必须推保护）：经典压单_抬单反转、紧贴着被吃掉的抬单

**统一模板**：出现背景 → 意义 → 级别 → 对开单的影响

**核心教训**：
1. 不要立刻反向 — 等 1-2 次 Sweep + 回到后方确认
2. L3 信号必须推保护（移动止损）
3. 区分大空单/大多单的不同含义

#### 领域 C: 期权与 GEX

**核心框架**：
- **GEX 正 → 震荡**：做市商 Long Gamma，逆势对冲（负反馈），价格被"钉"在区间
- **GEX 负 → 趋势**：做市商 Short Gamma，顺势对冲（正反馈），波动放大

**关键阈值**（来自 `组合 -ES.md` 和 `5.28.md`）：
- NQ 上 Net GEX/Total GEX > 0.45 才算健康正 Gamma
- Expiring GEX/Net GEX 在 0.3~0.8 时短期效应增强
- 月度 OPEX 日 Expiring GEX/Total GEX 可达 40%-75%+

#### 领域 D: 复盘实践

**覆盖时段**：2025.5.14–5.22（7 个交易日）+ 5.28（GEX 分析）

**复盘格式**：盘前关键位分级（L1-L4）→ 多空情景推演 → 盘中注意事项

**特色**：每条计划都包含 ZG/CW/PW 具体价位、三种情景（多/空/震荡）及各自的触发条件

## 经验总结

### 做得好的

1. **Mubu Exporter 扩展**是可行的导出方案，保护了所有文字内容
2. **Obsidian Vault** 结构简单清晰，按知识域分文件夹
3. **元笔记**（术语表、概念地图、MOC）是后续可维护性的关键
4. **双向链接**使零散的 28 篇笔记变成了一个互联网络

### 遗留问题

1. **图片全部丢失**：21 张原幕布图片（图表、截图）依赖 `document-image.mubu.com` CDN，未来可能完全下线
   - 建议：手动截图保存到本地 `attachments/` 文件夹，替换占位符
2. **量化标准缺失**：大量主观描述（"浅回撤""大成交量"）无数字锚定
   - 建议：在复盘笔记中逐步积累统计数据
3. **`震荡_结构组合.md`** 仍为空 — 震荡市是常见的亏钱场景，需要补充
4. **L1 模式缺失**：最强信号级别的具体模式尚未记录

### 可复用的迁移方法论

```
1. 导出：找到从旧工具批量导出 Markdown 的途径
2. 结构：按知识域分文件夹，每个文件夹放 MOC 索引
3. 诊断：全量阅读 → 分类问题（格式/内容/结构）
4. 清理 → 修复 → 增强 → 链接 → 归档 五步走
5. 元数据：术语表 + 概念地图 是知识库可维护性的基础设施
```
